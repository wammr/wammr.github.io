Structure of a Compiler

Lexical Analysis 词法分析

Goal: divides program text into “words” or “tokens”

理解每个word的性质

Parsing = Diagramming Sentences 语法分析

Goal: understand sentences structure

理解句子结构，形成一棵parse tree

Semantic Analysis 语义分析

Goal: understand meaning

但编译器只能做有限的语义分析，比如catch inconsistencies(类似前后变量类型的检查这类操作)

Optimization 优化

Automatically modify programs

Goal: Run faster. Use less memory.

例：X = Y * 0 在X和Y都是整型的情况下可以优化成X=0 （但若X和Y均为浮点型的话不能这么优化因为若X为NAN，NAN * 0 = NAN）

The Economy of Programming Language

Why are there so many programming languages?

很难设计出一种语言，其能满足多种相互冲突的需求。不同的需求对应不同语言的应用

  • scientific computing
    • good FP
    • good arrays
    • parallelism
    • —–>FORTRAN
  • business applications
    • persistence
    • report generation
    • data analysis
    • —–>SQL
  • systems programming
    • control of resources
    • real time constraints
    • —–> C/C++

Why are there new programming languages?

Widely-used languages are slow to change.

Easy to start a new language.

Languages adopted to fill a void.

已被人们广泛使用的旧的编程语言很难做出改变，因为用户这些年的使用习惯很难改变，所以旧的编程语言很难适应新需求，这就需要新语言的出现满足新的需求。

但新语言其实很难是全新的，它会与一些前代语言有相似之处。

What is a good programming language?

There is no universally accepted metric for language design.

没有一个普遍接受的共识说明到底什么是一个好的编程语言。

conclude

  • Application domains have conflicting needs.
  • Programmer training is the dominant cost for a programming language.

Cool Overview

这门课提出了一个Classroom-Object-Oriented-Language，简称COOL语言。下面对于Compliers的讲解都是基于为COOL语言实现一个编译器进行的。

  • Designed to be implementable in a short time

  • Cool compilers >> Cool programs

  • Give a taste of implementation of modern

    • Abstraction
    • Static typing
    • Reuse(inheritance)
    • Memory management
    • And more…

  • A complete compiler

    • Cool ->MIPS assembly language

  • PAs （后面的阶段完成）

    • Write a Cool program
    • Lexical analysis
    • Parsing
    • Semantic analysis
    • Code generation
    • Optimization *

具体的Cool 语言使用规则见：cool-manual.pdf (stanford.edu)

对Cool进行编译运行的过程：
